What No Shipping Product Combines (Yet)

Most observability AI tools run a confirmation cascade: findings → hypothesis → action. Every agent in the pipeline works to confirm the first guess, not challenge it. This implementation adds four mechanisms that break the cascade.

What most tools dovsWhat this adds
Hypothesis

Free-text assertions

Hypothesis statements are strings. No link to the evidence that supports them. Cannot be verified or cross-checked programmatically.

→
Mechanism 1

Provenance-bound claims

Every Claim must cite ≥1 Finding ID at construction time. Ungrounded claims are rejected with ValueError. The audit trail records each dropped claim.

Verification

No adversarial check

The reasoning agent builds a hypothesis. No agent is tasked with disproving it. The pipeline assumes the first hypothesis is correct and proceeds to action.

→
Mechanism 2

Adversarial Falsifier

A dedicated agent tries to disprove the hypothesis using the same findings. Returns {confirmed, contested, refuted}. A refuted verdict is a hard veto — autonomous action is blocked entirely.

Pre-action

Approve blindly

The approver sees "run this playbook." No prediction of what the system will look like after the action. The outcome is unknown until after execution.

→
Mechanism 3

Counterfactual simulation

Before the action is queued for approval, the pipeline predicts metric deltas with confidence bounds. The approver sees: "p99 latency will drop 65–80%, auth-service unaffected."

Confidence

Raw LLM score

Confidence = whatever the model says (e.g. "I'm 90% confident"). These scores are notoriously over-confident and uncalibrated. Brier score is never reported.

→
Mechanism 4

Calibrated confidence

Final = 0.4×(LLM score) + 0.4×(corpus prior) + 0.2×(falsification modifier). Brier score and ECE are measurable and reported. Formula is inspectable, not a black box.

Hypothesis accuracy: fraction of scenarios where a word from the true cause pattern appears in the hypothesis summary. Action precision: fraction where the recommended action type matches ground truth. Both are word-overlap metrics — intentionally approximate for a synthetic corpus.

Random baseline ECE

ECE is high (0.337) because DeterministicFake returns fixed confidence values that don't adapt to each scenario. With a real LLM that adjusts confidence per-scenario, ECE drops substantially. This is expected behaviour and is disclosed in docs/methodology.md.

Falsifier Performance

Falsifier precision
1.00
Falsifier recall
0.89

Precision 1.00: when the falsifier says "confirmed," the hypothesis is correct in every case. Recall 0.89: of all actually-correct hypotheses, 89% are correctly confirmed. The 11% gap reflects genuinely ambiguous scenarios where the falsifier downgrades to "contested."

Calibration Formula

final = clip(
  0.4 × raw_llm
  + 0.4 × corpus_prior
  + 0.2 × (1 − falsif_score)
, 0, 1)

Each component is auditable. The corpus prior comes from 100 ground-truth incidents. The falsification modifier moves confidence down when the falsifier finds contradictions. No black-box weighting.
